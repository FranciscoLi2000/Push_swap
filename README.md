# Push_swap

这个 **Push_swap 项目** 旨在通过实现一种**基于栈的排序算法**，帮助你掌握以下 **核心技能** 和 **算法思维**：

---

### **核心学习目标**
1. **算法设计与优化**
- 掌握经典排序算法（如快速排序、归并排序）的变体实现，针对栈操作的限制进行优化。
- 减少操作步骤（如合并旋转 `rr`、`rrr`），满足性能要求（100 个数 ≤700 步，500 个数 ≤5500 步）。

2. **数据结构操作**
- 深入理解栈（LIFO）的特性，灵活运用 `swap`、`push`、`rotate` 等操作实现排序逻辑。
- 管理双栈（stack a 和 stack b）的交互，设计数据迁移策略（如分块排序、分段合并）。

3. **C 语言编程实践**
- 动态内存管理（`malloc`/`free`），避免内存泄漏。
- 处理输入参数（字符串转整数、去重校验），实现健壮的错误处理（如非法输入输出 `Error\n`）。

4. **性能分析与复杂度控制**
- 评估算法时间复杂度，优化指令数以满足项目要求。
- 使用工具（如 `time`、`wc -l`）分析程序性能，验证大规模数据下的效率。

5. **代码规范与工程实践**
- 遵循 **Norminette** 规范（函数长度、缩进、命名），编写可维护代码。
- 编写高效 `Makefile`，支持自动化编译与清理。

6. **测试与验证**
- 实现 `checker` 程序（Bonus），验证指令序列的正确性。
- 设计测试用例覆盖边界条件（已排序、逆序、随机数据）。

---

### **关键技术点解析**
#### 1. 分块排序策略（针对大规模数据）
- **分块逻辑**：将栈 a 分为多个有序块，分块推送到栈 b，再逐步合并回栈 a。
- **优化指令数**：利用 `pb` 和 `ra` 结合，减少无效操作。

#### 2. 合并旋转优化
```c
	// 合并 ra + rb → rr
	if (need_rotate_a && need_rotate_b)
	{
		write(1, "rr\n", 3);
		steps++;
	}
	else
	{
		// 单独处理 ra 或 rb
	}
```
**关键点**：通过合并操作减少指令数量，提升效率。

#### 3. 输入处理与校验
```c
	// 检查输入是否为合法整数且无重复
	int	parse_args(int argc, char **argv, int **stack)
	{
		for (int i = 1; i < argc; i++)
		{
			if (!is_valid_int(argv[i]))
				return 0;
			if (is_duplicate(*stack, current_value))
				return 0;
		}
		return 1;
	}
```
**关键点**：使用 `atoi` 或自定义解析函数，结合哈希表去重。

### **工程实践与思维提升**
1. **模块化设计**
- 分离逻辑：输入解析、栈操作、算法核心、输出生成。
- 使用静态函数（`static`）隐藏内部实现，提升代码安全性。

2. **防御式编程**
- 处理极端输入（如单元素栈、空栈）。
- 防止栈溢出（如 `rotate` 操作时的边界检查）。

3. **性能调优**
- 对小规模数据（如 3-5 个数）硬编码最优解（如 `sa` 或 `ra`）。
- 对大规模数据采用分治策略，减少递归深度。

---

### **项目隐喻与深层价值**
- **“受限环境下的算法创新”**：在固定操作集中寻找最优解，培养解决问题的创造力。
- **“性能与资源的平衡”**：通过优化指令数，理解算法效率与资源消耗的权衡。
- **“从理论到实践”**：将排序算法理论转化为实际代码，强化工程实现能力。

### **通关要点**
1. **分步实现**
- 先实现基础排序逻辑（如插入排序），再优化为高效算法（如快速排序变体）。
- 逐步添加 `checker` 功能（Bonus），验证每一步的正确性。

2. **测试覆盖**
- 使用脚本生成随机数据测试（如 100/500 个数），统计指令数。
- 对比 `checker` 输出与预期结果，确保 100% 正确性。

3. **代码优化**
- 减少重复代码（如封装 `swap`、`rotate` 操作为函数）。
- 使用指针而非数组下标提升性能。
