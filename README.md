# Push_swap 项目解析

## 项目概述
**Push_swap** 是一个算法优化项目，目标是通过有限的操作指令对两个栈（a 和 b）中的整数进行排序，并确保操作次数最少。项目要求使用 C 语言实现，核心挑战在于设计高效的排序算法，同时遵守严格的编码规范和性能要求。

---

## 核心规则
1. **初始状态**：
- 栈 a 包含随机数量的整数（正数/负数）。
- 栈 b 为空。
2. **允许的操作**：
- **交换**：`sa`（交换 a 的前两个元素）、`sb`（交换 b 的前两个元素）、`ss`（同时执行 sa 和 sb）。
- **推送**：`pa`（将 b 的顶部元素推入 a）、`pb`（将 a 的顶部元素推入 b）。
- **旋转**：`ra`（a 的所有元素上移一位）、`rb`（b 的所有元素上移一位）、`rr`（同时执行 ra 和 rb）。
- **逆旋转**：`rra`（a 的所有元素下移一位）、`rrb`（b 的所有元素下移一位）、`rrr`（同时执行 rra 和 rrb）。
3. **目标**：
使用最少操作将栈 a 按升序排列。

---

## 项目要求
### 主程序 `push_swap`
- **输入**：接受一组整数作为栈 a 的初始状态（第一个参数为栈顶）。
- **输出**：生成最短的操作指令序列，每条指令用换行分隔。
- **错误处理**：
输入无效（非整数、重复值、溢出）时输出 `Error\n`。
- **性能要求**：
- **100 个随机数**：不超过 700 步。
- **500 个随机数**：不超过 5500 步（用于满分和 Bonus）。

### 编码规范
- 使用 C 语言，遵循 **Norminette** 规范。
- 禁止全局变量、内存泄漏。
- 提供标准 `Makefile`，包含 `all`, `clean`, `fclean`, `re` 规则。

---

## 算法设计思路
### 关键挑战
- **最小化操作次数**：需结合多种操作（如旋转与逆旋转的灵活组合）。
- **分阶段处理**：
1. **预处理**：将部分元素移动到栈 b，减少后续排序复杂度。
2. **合并排序**：利用栈操作特性，分块合并有序序列。
3. **优化路径**：通过动态规划或贪心策略选择当前最优操作。

### 常见策略
- **分块法**：将栈 a 分成多个块，按阈值将元素推入栈 b，再逐个排序合并。
- **模拟退火**：尝试不同操作组合，评估步数并保留最优解。
- **快速排序变体**：以某个基准值为中心，分割栈元素到 a 和 b。

## 附加任务：Bonus 部分
### 程序 `checker`
- **功能**：验证 `push_swap` 生成的指令是否能正确排序栈。
- **输入**：接受初始栈和指令序列。
- **输出**：执行指令后若栈 a 有序且栈 b 为空，输出 `OK\n`，否则 `KO\n`。
- **错误处理**：检测无效指令或输入时输出 `Error\n`。

### 实现要点
- 需模拟栈操作，跟踪指令执行后的状态。
- 严格匹配操作格式（如大小写敏感、非法指令）。

---

## 测试与优化
1. **边界测试**：
- 空输入、单个元素、完全逆序、已排序的情况。
- 大数（接近 INT_MAX/MIN）和重复值检测。
2. **性能测试**：
- 使用脚本生成随机数，统计平均操作次数。
- 对比不同算法策略的效率（如分块大小对步数的影响）。
3. **内存检查**：
- 使用 `valgrind` 确保无内存泄漏。

---

## 总结
Push_swap 项目的核心是**算法优化能力**与**C 语言工程实践**的结合。开发者需深入理解栈操作特性，设计灵活的分阶段策略，并通过大量测试验证性能。Bonus 部分进一步考验对问题全流程的掌控能力。最终目标是平衡代码简洁性、执行效率和规范性。
